# Деплой на RelaxDev с нуля

Один проект на RelaxDev = **сайт + API + PostgreSQL**.  
Локально у вас по-прежнему `start.bat` → http://localhost:3000.

> **Важно:** отдельной кнопки «создать базу» в главном меню нет. Сначала создаёте **проект** из GitHub, потом **внутри проекта** — вкладка **«База данных»**.

---

## Что понадобится

1. Аккаунт на [relaxdev.ru](https://relaxdev.ru/) (у вас уже есть)
2. Аккаунт на [GitHub](https://github.com) (бесплатно)
3. 30–40 минут времени

---

## Часть 1. Загрузить код на GitHub

### 1.1. Установите Git (если нет)

Скачайте: https://git-scm.com/download/win  
При установке оставьте настройки по умолчанию.

### 1.2. Создайте репозиторий на GitHub

1. Войдите на github.com  
2. Нажмите **+** → **New repository**  
3. Имя, например: `svjezhij-urozhaj`  
4. **Private** или Public — как хотите  
5. **Не** ставьте галочки README / .gitignore  
6. **Create repository**

### 1.3. Отправьте проект с компьютера

Откройте **PowerShell** в папке проекта (`D:\Fort\Project`):

```powershell
cd D:\Fort\Project
git init
git add .
git commit -m "Подготовка к деплою на RelaxDev"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/svjezhij-urozhaj.git
git push -u origin main
```

`ВАШ_ЛОГИН` — ваш логин GitHub. При `git push` браузер попросит войти в GitHub.

> Файлы `.env` с паролями в git не попадут — они в `.gitignore`.

---

## Часть 2. Создать проект на RelaxDev

1. Войдите в [панель RelaxDev](https://relaxdev.ru/)  
2. Нажмите **«Добавить проект»** (или **Новый проект**)  
3. Подключите **GitHub**, если ещё не подключали  
4. Выберите репозиторий `svjezhij-urozhaj`  
5. Ветка: **`main`**

### Настройки при создании (важно!)

| Параметр | Значение |
|----------|----------|
| **Автоопределение стека** | **ВЫКЛ.** |
| **Сборка** | **Свой Dockerfile** |
| Путь к Dockerfile | `Dockerfile` (в корне репозитория) |
| Корень (rootDir) | `/` — **не** `frontend` |
| Порт | `8000` |
| Автодеплой | **вкл.** |

> Если оставить автоопределение, RelaxDev решит что это Next.js и соберёт **чужой** Dockerfile — ошибка `package.json: not found`.

### Переменные окружения (добавьте сразу)

Вкладка **Переменные** / **Environment** (см. `deploy/relaxdev.env.example`):

| Переменная | Значение |
|------------|----------|
| `SECRET_KEY` | длинная случайная строка, минимум 32 символа |
| `CORS_ORIGINS` | `*` |
| `SERVE_FRONTEND` | `1` |
| `RUN_SEED` | `1` (только **первый** деплой) |
| `PORT` | `8000` |

На **пробном тарифе** `DATABASE_URL` **не указывайте** — приложение само включит SQLite.

6. Нажмите **«Деплой»**

---

## Часть 3. База данных

### Пробный тариф (PostgreSQL недоступен)

Ничего делать не нужно. После успешной сборки сайт работает на **SQLite**.

Ограничение: при полной **пересборке** контейнера заказы и товары могут сброситься. Для постоянного магазина — платный тариф с PostgreSQL или [VPS](./DEPLOY.md#3-api-на-vps-запасной-и-надёжный-вариант).

### Платный тариф (есть PostgreSQL)

1. Откройте проект → вкладка **«База данных»**  
2. **Создать PostgreSQL** в один клик  
3. `DATABASE_URL` появится в переменных автоматически  
4. **Redeploy**

### Если вкладки «База данных» нет

Это **нормально на пробном тарифе**. Используйте SQLite (см. выше) или [VPS](./DEPLOY.md#3-api-на-vps-запасной-и-надёжный-вариант).

### Тумблер «Сборка с доступом к БД»

Для нашего проекта **не обязателен**. Он нужен Next.js-сайтам, которые читают БД **во время сборки**. Наш магазин обращается к БД только **после запуска** контейнера.

## Часть 4. Проверка

1. У проекта появится ссылка `https://имя-проекта.relaxdev.ru`  
2. Откройте в браузере — должен открыться **магазин**, не JSON  
3. Проверка API: `https://ваш-url/health` → `{"status":"ok"}`  
4. Вход админа (заполняется при `RUN_SEED=1`):  
   - телефон `+79001234567` или email `admin@example.com`  
   - пароль `admin123`  

**После первого входа смените пароль админа в профиле.**  
После успешного деплоя **`RUN_SEED` можно удалить** или поставить `0`.

## Часть 5. Обновление сайта

После изменений в коде:

```powershell
cd D:\Fort\Project
git add .
git commit -m "Описание изменений"
git push
```

При включённом автодеплое RelaxDev пересоберёт проект сам. Иначе — **Redeploy** в панели.

## Часть 6. Свой домен (по желанию)

В проекте: вкладка **«Домены»** → добавить `ваш-домен.ru` → прописать DNS у регистратора (A/CNAME как подскажет RelaxDev). SSL выпустится автоматически.

## Если что-то пошло не так

### Ошибка `package.json: not found` / сборка Next.js (node:22-alpine)

В логе видно `NEXTAUTH_SECRET`, `COPY package.json` — RelaxDev **не использует ваш Dockerfile**, а пытается собрать как Next.js.

**Исправление:**

1. Проект → **Настройки** → **Сборка** / **Стек**
2. **Выключите** «Автоопределение стека»
3. Выберите **«Свой Dockerfile»**
4. Путь: `Dockerfile`, корень: `/`
5. Порт: `8000`
6. **Redeploy** → «Без кэша Docker» (если есть галочка)

В логе должно быть: `FROM node:20-alpine AS frontend-build` и `FROM python:3.12-slim`.

### `github account not linked or token is missing`

RelaxDev **не видит** ваш GitHub — код с репозитория не скачивается.

**Шаг 1 — привязать GitHub (обязательно до деплоя):**

1. RelaxDev → **иконка профиля** (правый верхний угол) → **Настройки** / **Профиль**
2. Раздел **Git** / **Интеграции** / **Провайдеры**
3. Нажмите **«Подключить GitHub»** → откроется GitHub
4. Войдите в GitHub → **Authorize RelaxDev**
5. В доступе к репозиториям выберите **All repositories** или конкретный `svjezhij-urozhaj`

**Шаг 2 — проверить проект:**

1. Откройте проект → **Настройки** → **Источник деплоя**
2. Убедитесь, что выбран **GitHub** и правильный репозиторий + ветка `main`
3. **Redeploy**

**Шаг 3 — если снова та же ошибка:**

- Отключите GitHub в настройках RelaxDev и **подключите заново**
- Проверьте, что код **уже на GitHub** (`git push` прошёл успешно)
- Репозиторий **не пустой** (есть коммит с `Dockerfile` в корне)

**Запасной вариант — Gitverse (российский Git):**

RelaxDev хорошо работает с [Gitverse](https://gitverse.ru/):

1. Подключите Gitverse в профиле RelaxDev
2. В проекте: **Миграция** — скопировать репозиторий с GitHub в Gitverse в один клик
3. **Источник деплоя** → переключить на Gitverse → Redeploy

### Ошибка сборки / «Can't connect to database»

- Создана ли БД во вкладке **«База данных»** проекта?  
- Есть ли `DATABASE_URL` в **Переменных**?  
- Сделали **редеплой** после создания БД?

### Белый экран

- Проверьте `SERVE_FRONTEND=1`  
- Откройте **Логи** деплоя в панели

### Не удаётся войти

- При первом деплое был `RUN_SEED=1`?  
- После добавления БД был **редеплой**?

### Картинки пропали после redeploy

Диск контейнера может очищаться. Позже можно подключить **файловое хранилище** RelaxDev / [IziPost](https://izipost.ru/) (вкладка хранилища в проекте).

### Нужна помощь

Напишите в поддержку RelaxDev: «Деплою Docker + FastAPI + PostgreSQL из GitHub, нужна вкладка База данных в проекте».
