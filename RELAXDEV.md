# Деплой на RelaxDev с нуля

Один проект на RelaxDev = **сайт + API + PostgreSQL**.  
Локально у вас по-прежнему `start.bat` → http://localhost:3000.

---

## Что понадобится

1. Аккаунт на [relaxdev.ru](https://relaxdev.ru/) (у вас уже есть)
2. Аккаунт на [GitHub](https://github.com) (бесплатно)
3. 30–40 минут времени

---

## Часть 1. Загрузить код на GitHub

### 1.1. Установите Git (если нет)

Скачайте: https://git-scm.com/download/win  
При установке оставьте настройки по умолчанию.

### 1.2. Создайте репозиторий на GitHub

1. Войдите на github.com  
2. Нажмите **+** → **New repository**  
3. Имя, например: `svjezhij-urozhaj`  
4. **Private** или Public — как хотите  
5. **Не** ставьте галочки README / .gitignore  
6. **Create repository**

### 1.3. Отправьте проект с компьютера

Откройте **PowerShell** в папке проекта (`D:\Fort\Project`):

```powershell
cd D:\Fort\Project
git init
git add .
git commit -m "Подготовка к деплою на RelaxDev"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/svjezhij-urozhaj.git
git push -u origin main
```

`ВАШ_ЛОГИН` — ваш логин GitHub. При `git push` браузер попросит войти в GitHub.

> Файлы `.env` с паролями в git не попадут — они в `.gitignore`.

---

## Часть 2. PostgreSQL на RelaxDev

1. Войдите в [панель RelaxDev](https://relaxdev.ru/)  
2. Найдите раздел **Базы данных** / **PostgreSQL** (название может быть «Создать БД»)  
3. Создайте новую базу, например имя `shop`  
4. Сохраните строку подключения — это **`DATABASE_URL`**  
   - вид: `postgresql://user:pass@host:5432/shop`  
   - платформа может выдать её после создания кластера

Если не найдёте — напишите в поддержку RelaxDev (у них заявлена помощь с деплоем 24/7).

---

## Часть 3. Создать проект (деплой)

1. В панели RelaxDev: **Новый проект** / **Deploy** / **Импорт из Git**  
2. Подключите **GitHub** (разрешите доступ к репозиторию)  
3. Выберите репозиторий `svjezhij-urozhaj`  
4. Ветка: **`main`**

### Важные настройки проекта

| Параметр | Значение |
|----------|----------|
| Тип / стек | **Docker** (если спрашивают) |
| Корень репозитория | `/` (корень, где лежит `Dockerfile`) |
| Dockerfile | `Dockerfile` |
| Порт | `8000` |
| Health check | `/health` |

### Переменные окружения

В разделе **Environment** / **Переменные** добавьте (см. `deploy/relaxdev.env.example`):

| Переменная | Значение |
|------------|----------|
| `DATABASE_URL` | строка из шага «PostgreSQL» (часто подставляется при привязке БД) |
| `SECRET_KEY` | любая длинная случайная строка, например `my-super-secret-key-2026-change-me` |
| `CORS_ORIGINS` | `*` |
| `SERVE_FRONTEND` | `1` |
| `RUN_SEED` | `1` (только **первый** деплой — заполнит товары и тестовых пользователей) |
| `PORT` | `8000` |

После первого успешного деплоя **`RUN_SEED` можно удалить** или поставить `0`, чтобы при перезапуске не дублировать данные.

5. Нажмите **Deploy** / **Запустить**

Сборка занимает **5–15 минут** (собирается фронт и бэкенд).

---

## Часть 4. Проверка

1. RelaxDev выдаст ссылку вида `https://....relaxdev.ru`  
2. Откройте в браузере — должен открыться **магазин**, не JSON  
3. Проверка API: `https://ваш-url/health` → `{"status":"ok"}`  
4. Вход админа (из seed):  
   - телефон `+79001234567` или email `admin@example.com`  
   - пароль `admin123`  

**После первого входа смените пароль админа в профиле.**

---

## Часть 5. Обновление сайта

После изменений в коде:

```powershell
cd D:\Fort\Project
git add .
git commit -m "Описание изменений"
git push
```

RelaxDev обычно **сам пересобирает** проект после push (если включён автодеплой). Иначе — кнопка **Redeploy** в панели.

---

## Часть 6. Свой домен (по желанию)

В панели проекта: **Домены** → добавить `ваш-домен.ru` → прописать DNS у регистратора (A/CNAME как подскажет RelaxDev). SSL выпустится автоматически.

## Если что-то пошло не так

### Белый экран или ошибка сборки

- Откройте **Логи** / **Build logs** в панели  
- Частая причина: не задан `DATABASE_URL`

### Сайт открывается, но «не удаётся войти»

- Убедитесь, что `RUN_SEED=1` был при первом деплое  
- Или в консоли RelaxDev (если есть shell) выполните seed вручную  

### Загруженные в админке картинки пропали после redeploy

На бесплатном плане диск контейнера может очищаться. Решения:

- подключить **хранилище / volume** в панели RelaxDev (если доступно)  
- позже — [IziPost](https://izipost.ru/) для файлов  

### Нужна помощь

Напишите в поддержку RelaxDev — укажите лог сборки и что используете `Dockerfile` из репозитория.

## Локальная разработка (как раньше)

```bat
start.bat
```

Сайт: http://localhost:3000  
API: http://localhost:8000/docs  

PostgreSQL локально (опционально):

```powershell
docker compose up -d db
# backend\.env → DATABASE_URL=postgresql+psycopg://shop:shop@localhost:5432/shop
python seed.py --reset
```
